ЦБК - цифро-буквенный код

1. для парсинга (и вообще внутри раскодировщика) используются имена соответствующие символической букве(ам)
2. для передачи вовне в xml используется дескриптор из 5-7 цифр.
4. при наличии однозначного отображение в дескриптор bufr (и понятие величины, и единицы измерения), используется номер дескриптора bufr для 5 младших цифр, две старшие равны '0' (можно опустить)
5. в случае нескольких независимых параметров с разным обозначением в ЦБК и одним номером в bufr:
   - если нет необходимости для создания своего кода для каждого параметра при сохранении (параметры не связаны ни с какими другими параметрами (напр, CL, CM, CH); все связанные параметры имеют одинаковое количество значение (напр, значения по уровням)), все значения будут храниться под одним номером (multihash);
   - если необходимо создать новый номер для сохранения (в bufr только общее понятие (напр. продолжительность), передаётся группа связанных параметров, часть их которых пересекается по номеру bufr с уже имеющимися независимыми (напр, С_)), тогда необходимо указать номер группы из двух цифр. Для этого в описании дескрипторов descriptors.xml для указания группы необходимо задать тег  group="XX", где XX=01-99. При сохранении будет использоваться полный номер дескриптора из 7 цифр: 2 старшие - номер группы, 5 младших - номер дескриптора в bufr

302033 - видимость
